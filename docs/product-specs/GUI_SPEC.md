# GUI规格

## 主要用户结果

用户可以通过中国证券/基金代码或名称搜索标的，切换当前证券，查看实时和历史价格走势，理解当前使用的策略，查看概率化预期走势，并获得可解释的操作状态。

## 主界面布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 搜索 │ 数据健康 │ 行情时间 │ 设置                            │
├──────────────┬───────────────────────────┬───────────────────┤
│ 自选列表     │ 价格折线 / K线             │ 当前策略          │
│ 指数         │ 成交量 / 指标              │ 预期走势          │
│ 最近访问     │ 信号标记 / 预测区间        │ 操作与风险        │
├──────────────┴───────────────────────────┴───────────────────┤
│ 市场 │ 策略 │ 回测 │ 模拟账户 │ 风险 │ 知识中心             │
└──────────────────────────────────────────────────────────────┘
```

参考线框图：`../../assets/gui_wireframe.png`。

## 搜索与证券切换

- 输入支持代码、名称和别名；本地匹配使用300毫秒防抖。
- 结果显示代码、名称、资产类型、交易所、最新状态和行情年龄。
- 键盘上下键改变高亮结果，回车确认。
- 确认选择后启动一次原子化切换事务：
  1. 增加 `selection_generation`；
  2. 取消旧请求和订阅；
  3. 显示新证券的本地缓存历史数据；
  4. 异步补齐缺失数据并订阅实时行情；
  5. 重新计算指标、策略、预测和风险；
  6. 丢弃携带旧代次标记的所有结果。
- 所有面板使用的证券和 `as_of` 时间必须一致。

## 图表

必需模式：

- 分时折线；
- 数据权限允许时支持1/5/15/30/60分钟；
- 日线/周线/月线；
- 5日、1月、3月、6月、1年、3年、5年和自定义范围。

必需交互：

- 平移、缩放、十字光标、悬停提示和范围切换；
- 实时数据增量更新，不得全图重绘；
- 复权模式：不复权/前复权/后复权，并明确显示当前模式；
- 可选叠加MA/EMA、成交量、指标、信号和预测区间；
- 可点击买入/卖出/止损标记，查看当时解释和后续实际结果。

预测可视化：

- 使用分位数区间、可信区间或分布摘要，不绘制一条确定性的未来价格线；
- 预测图层必须与真实价格使用不同视觉样式；
- 显示预测周期、覆盖概率、模型版本、时点时间和警告“预测区间，不代表确定未来价格”。

## 当前策略面板

始终显示：

- 策略名称、ID和版本；
- 预测周期和再平衡频率；
- 市场状态；
- 简短逻辑说明；
- 适用条件和失效条件；
- 默认只读的关键参数；
- 模型、规则和数据状态。

## 预期走势面板

允许的分类：

- 强势上涨；
- 震荡上涨；
- 横盘；
- 震荡下跌；
- 强势下跌；
- 不明确/不交易。

显示方向概率、预期收益分位数、预期最大不利变动/回撤、置信度/校准状态、正向驱动、负向驱动和样本充分性。

## 操作面板

允许状态：

`BUY_CANDIDATE`、`ADD_CANDIDATE`、`HOLD`、`REDUCE`、`SELL`、`WATCH`、`ABSTAIN`。

分别表示：买入候选、加仓候选、持有、减仓、卖出、观察、不交易。

显示等级、有效期、触发条件、失效/退出条件、仓位上限和风险原因。`ABSTAIN` 必须说明原因属于数据、规则、模型不确定性、流动性、风险预算或预期价值不足中的哪一类。

## UI状态机

至少包括：

- 空闲；
- 正在搜索；
- 正在加载缓存历史数据；
- 正在获取远程历史数据；
- 正在连接实时行情；
- 实时运行；
- 正在计算策略；
- 数据陈旧；
- 被数据质量阻断；
- 历史数据不足；
- 模型分布外；
- 回测运行中/取消中/失败/完成。

颜色不得是唯一的状态表达方式。关键数据陈旧和信号阻断必须在不打开二级面板的情况下持续可见。
