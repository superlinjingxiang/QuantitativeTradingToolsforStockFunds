# 中国市场与基金规则

## 原则

市场规则是按日期生效的数据，不是永久常量。规则引擎必须按交易所、资产类型、证券身份和交易日期解析行为。每条规则必须包含来源、生效日期、审核状态和版本。

## 资产隔离

| 资产 | 分析/执行价格 | 日内行为 | 必须特殊处理的事项 |
|---|---|---|---|
| A股 | 交易所成交价格 | 普通股票遵循适用的可卖数量和回转交易限制 | 不同板块涨跌停、停牌、公司行为、手数和最小价位 |
| 场内ETF/LOF | 交易所成交价格 | 是否允许日内回转取决于产品和规则 | 产品分类、价差、流动性、折溢价、跟踪误差 |
| 场外公募基金 | 正式披露净值 | 不是交易所盘中交易工具 | 截止时间、确认、申购赎回费、净值披露 |
| 指数 | 计算得到的指数点位 | 除非映射到产品，否则不可交易 | 仅用于基准和市场状态 |

## 最低规则能力

- 交易日历和交易时段；
- 订单方向/类型、价格与最小价位校验；
- 手数与最小数量；
- 可卖数量和适用的回转交易限制；
- 涨跌停制度和特殊期间；
- 停牌与复牌；
- 按生效日期和方向计算的佣金、税费与过户费；
- 公司行为以及现金/股份调整；
- 退市和历史证券状态；
- 基金确认与正式净值语义；
- 特定产品的日内交易资格。

## 规则解析契约

```python
rule = rule_repository.resolve(
    exchange=security.exchange,
    asset_type=security.asset_type,
    security_id=security.id,
    trade_date=event.trade_date,
)
```

如果无法解析必需规则，回测或新信号必须以 `RuleMissing` 关闭式失败。存在时点主数据时，不得仅根据证券代码前几位猜测规则。

## 对回测的影响

- 价格触及涨跌停不代表一定成交。
- 停牌证券不能产生正常可交易K线或成交。
- 总持仓数量与当前可卖数量是不同概念。
- 公司行为必须影响现金与股份；仅使用复权价格不足以完成组合记账。
- 历史证券池在适用时必须保留退市、停牌、ST状态和指数历史成分。
- 正式基金净值与盘中估算净值必须使用不同字段和数据集。

## 治理流程

规则来源更新时必须完成：

1. 保存权威来源；
2. 审核生效日期；
3. 更新迁移和版本；
4. 为边界日期增加单元测试；
5. 对受影响回测执行回归测试；
6. 如果行为改变，增加审计记录和决策记录。
