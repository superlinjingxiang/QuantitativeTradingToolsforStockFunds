## 近期整合摘要

### 运行与加载

- 默认桌面界面改为 Vue 3 + TypeScript + ECharts，Electron 负责桌面窗口，FastAPI 复用现有 Python 行情、策略、预测、回测和账户服务。
- Redis 只作为可选热缓存；Redis 不可用时自动使用进程内缓存，历史 K 线继续使用 Parquet。本地缺少 Redis 不会阻止一键启动。
- 页面先恢复最近一次成功结果，再在后台刷新；上游失败时保留上一份数据并显示 `STALE`/`DEGRADED`，不会把缓存伪装成实时行情。
- 修复 Electron 生产构建启动黑屏、旧请求覆盖新标的和自动刷新重复并发等问题。

### 行情与界面

- 输入代码后回车直接分析；自选列表和历史浏览记录保存在本机，重启后继续显示。
- 市场指数和荐股池加入缓存、启动预取和失败保留，减少首次进入或公开数据源短暂断开时的空白状态。
- 当前价格增加方向箭头，遵循中国市场习惯：上涨红色向上、下跌绿色向下。
- 预测区间使用区别于历史价格的虚线、色带和预测端点，并与行情共用日期轴和价格轴；回测信号同样按当前图表周期和日期对齐。
- 浅色主题重新设置文字、表格、按钮、悬停和选中态的对比度；行情图绘图区保持高对比暗色背景。
- 页面支持整体纵向滚动，右侧四块决策卡、底部证据区和长文本可以完整查看。

### 荐股与账户

- 荐股池只返回普通 A 股账户可买的 A 股、场内 ETF、LOF 和境内跨境 ETF；不直接推荐 QQQ、SPY 等境外账户代码。
- 候选按总分降序展示，标明 T+0/T+1、触发条件、止损/止盈、仓位上限和数据缺失原因；点击候选可进入同一标的分析流程。
- 手动账户支持计划资金、现金、数量、成本价和风险偏好；账户建议只消费当前策略最终信号和仓位上限，不另写第二套策略，也不读取券商账户或执行下单。

### 策略与证据边界

- A 股短线策略完成 V2-V7 的风险调整、成本压力、目标波动、反追涨、次日开盘、T+1 和市场环境约束；四组固定 A 股池仍未形成通用 PASS，因此新增仓位继续失败关闭。
- ETF 组合 V9 候选在全样本有正收益和正超额，但最终时间留出只有 1 个完整滚动窗口，状态保持 `WATCH`，尚未进入模拟盘和单标的执行链。
- 预测区间升级为 `forecast.similar_regime_interval.v2`，清除 21 日标签重叠和近端训练标签泄漏，并把覆盖率、下破率和三分类 Brier 纳入 DecisionHub 门禁。
- 当前产品仍是研究、回测和模拟验证工具；界面中的“观察”“仅研究观察”和“暂不交易”是证据门禁结果，不是占位文案，也不代表收益承诺。

## 本轮策略研究

- 新增 `strategies.etf_rotation_validation`，对固定十ETF组合执行按时点回测、目标波动仓位、基础/压力成本和滚动前推验证。
- 新增 `strategies.etf_rotation_lab`，可从默认多数据源拉取真实历史并复现全样本和最后25%时间留出。
- ETF候选固定为252日正绝对动量、每21日选择最多2只、20%目标年化波动、下一交易日开盘执行。
- 新增至少3个完整滚动窗口硬门槛。最后25%虽收益+43.64%、超额+16.78%，但只有1折，因此结果为WATCH而不是PASS。
- 记录并拒绝股票5日无跟随退出和股票横截面动量候选；正式A股策略继续使用V7。

## 边界

- V9 组合接入没有重写当前单标的信号公式或概率预测；手动账户只增加对同一 DecisionHub 最终门禁和组合仓位参照的消费，不产生第二套策略。
- ETF组合候选尚未进入模拟盘，未覆盖容量、冲击成本和ETF品种交易制度，因此即使历史留出收益为正，也不能升级为正式交易许可。
- 任何回测结果都不构成未来收益承诺。

## 预测区间校准 V2

- 修复21日预测逐日校准造成的标签重叠和有效样本虚高。
- 历史评估时隔离完整持有期，训练只使用当时已经兑现的标签，消除近端未来标签泄漏。

详细方法、逐标的结果和复现命令见 `research/FORECAST_INTERVAL_VALIDATION_V2_2026-07-17.md`。校准通过只说明预测证据更可信，不代表策略能够保证盈利。

## ETF组合证据与账户同链联动

- `AnalysisReport`新增结构化`portfolio_strategy_evidence`，固定十ETF可展示V9验证状态、信号/执行日、入选代码、横截面排名、动量、研究仓位和滚动折证据。
- 当前组合快照复用V9回测同一路径，不另写一套当日排名公式；仍为前收盘信号、次日开盘执行。
- DecisionHub增加组合策略门禁：WATCH、缓存旧值或未入选只允许研究观察，不能成为新增仓位许可。
- 手动账户改为同时读取当前策略和最终DecisionHub门禁。空仓服从最终门禁；已有持仓不会因为证据不足的WATCH被误判为强制清仓，明确SELL/REDUCE仍保留风险优先级。
- 右侧当前策略、预期走势和操作风险增加组合参照；明确横截面组合排名不替代单标的方向概率。
- 固定十ETF证据默认保留30分钟，刷新失败时显示旧值状态；分析缓存Schema升级到v3。
