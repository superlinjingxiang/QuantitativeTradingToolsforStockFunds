# 完成定义（Definition of Done）

只有所有适用条件均满足，任务才算完成。

## 实现

- 行为满足关联的需求编号和验收编号。
- 保持架构边界。
- 实现错误、取消、降级和不交易路径。
- 不引入密钥、未授权数据或真实交易能力。

## 验证

- 新增或更新相关单元、集成、回归和GUI测试。
- 配置完成后，测试、格式化、代码检查和类型检查全部通过。
- 已审查金融时序、规则日期、陈旧数据、流动性和可复现性风险。
- 已实际操作用户可见行为，不得仅根据代码推断。

## 文档与追踪

- 事实源文档与实现一致。
- `docs/TRACEABILITY.md` 和 `spec/requirements.yaml` 已更新。
- API、契约和Schema随代码同步变更。
- 重大决策或新增依赖记录在 `docs/DECISIONS.md`。
- 活动执行计划的进度和决策日志为最新；已完成计划已归档。

## 交付

- 不存在高严重度安全或数据完整性问题。
- 日志/指标能够暴露重要故障模式，且不泄漏凭据。
- 可以根据记录的版本、快照和随机种子复现结果。
- 适用时，变更可以回滚或安全重试。
