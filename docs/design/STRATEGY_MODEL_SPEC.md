# 策略与模型规格

## 策略流水线

```text
时点数据
  -> 数据质量门禁
  -> 市场状态
  -> 证券池/可交易性过滤
  -> 指标和因子
  -> 原始策略评分/信号
  -> 校准后的预测分布
  -> 风险与组合约束
  -> 最终操作或ABSTAIN（不交易）
  -> 解释与审计记录
```

参考图：`../../assets/strategy_pipeline.png`。

## 策略决策中枢定位

产品后续优化的核心不是继续堆叠界面组件，而是把策略、预测、风控、回测和模拟盘证据收敛到一个可复现的策略决策中枢。GUI负责让用户查看数据、理解原因和复核证据；策略决策中枢负责回答“当前标的是否值得做、怎么做、做到什么仓位、什么条件下失效、历史上这种判断是否有正期望”。

决策中枢的目标是提高长期风险调整后收益和决策质量，不承诺单笔交易盈利。任何建议都必须能够追溯到输入数据、模型版本、策略版本、规则版本、风险检查和验证证据。

## 当前优先级：先证明有效，再优化界面

TASK-027优先关注三件事：

- 策略到底能不能赚钱：必须以扣除成本后的最终样本外、滚动前推、基准比较和风险调整收益证明正期望。
- 回测是否可信：必须审计未来数据泄漏、规则时点、成本/滑点/容量压力、参数敏感性和过拟合风险。
- 模拟盘是否能验证：必须记录信号到可得报价、模拟成交、滑点偏差、漏单/重复信号、延迟和异常恢复。

Electron/Web前端可以作为后续产品体验方向，但不得替代上述证据，也不得使任何缺证据策略进入执行候选。

## MVP策略

### ETF中期轮动

- 证券池：经过批准且流动性良好的宽基、行业/主题、黄金、债券和类现金ETF。
- 频率：每日计算；每周或满足条件时调仓。
- 假设：相对/绝对动量、趋势、波动率控制、分散化和相关性约束。
- 输出：选中资产、目标权重、现金权重、换手率和风险解释。
- 执行会计：保留现金和已有持仓，按执行日开盘名义金额与目标权重之间的差额交易；目标组合不变时不得全卖全买。
- 证据：每次调仓记录实际权重变化、换手和费用，容量审计与DecisionHub必须复用同一交易事件。

### 固定十ETF组合候选 V10

- 固定池、252日正绝对动量、21日调仓、最多2只和20%目标波动参数沿用V9，不因最终留出结果重新调参。
- 信号使用前一交易日及以前收盘信息，下一交易日开盘执行。
- 基础往返成本15bp，压力往返成本45bp；费用按实际交易差额扣除。
- 全样本、最后20%、25%和30%时间切片必须分别报告，不得用单一正收益切片覆盖滚动折不足。
- 最终留出完整252日滚动折少于3时保持`WATCH`，不能进入`PAPER_READY`。

### V11双周期确认研究开关（未晋级）

- `DUAL_HORIZON_CONSENSUS` 要求252日和126日绝对动量同时为正，并按两个周期的年化动量均值排名。
- 该模式只能由研究命令显式选择；`EtfRotationBacktestConfig()`和桌面应用默认必须保持`SINGLE_HORIZON`。
- 固定十ETF开发段中，双周期候选虽然把最大回撤从-21.54%改善到-11.66%，但净收益、Sharpe、滚动正收益比例和换手均变差；最后30%留出相对等权基准为负。
- 候选状态为“拒绝晋级”。不得在右侧四模块、DecisionHub或账户评估中把它显示为当前策略，也不得根据本次留出结果继续调确认窗口。

### V12跳过最近一月研究开关（开发段未晋级）

- `SKIP_RECENT_MONTH`把252日动量排名终点固定为`t-21`，公式为`close[t-21] / close[t-252] - 1`；21日跳过窗口短于形成窗口。
- 跳过只影响横截面排名。当前63日波动率继续使用最新可见数据控制仓位，不能因为跳过排名窗口而忽略近期风险。
- 固定十ETF开发段中，候选最大回撤和Sharpe略有改善，但净收益由+46.91%降为+45.52%，45bp压力收益由+44.41%降为+43.07%，未达到预注册门槛。
- 失败后不计算候选全样本、最终20%/25%/30%时间留出、容量或当前持仓，也不搜索其他跳过窗口。`EtfRotationBacktestConfig()`和应用链路继续默认`SINGLE_HORIZON`。
- 该实验来自经典股票prior (2-12) momentum的候选动机，但美国股票因子定义不能直接作为中国ETF有效性证明。

### ETF盈利验证算法

- 第一版实现为 `strategies.profit_validation`，用于回答“这个策略历史样本外有没有赚钱证据”，不承诺未来收益。
- 用户入参：周期 `1m/3m/6m/1y`，以及每年最大交易次数。交易次数按完成的样本外进出场约束，不按界面点击次数计算。
- 特征：短周期动量、长周期动量、价格相对趋势均线、年化波动率、滚动回撤、成交量确认和流动性确认。
- 入场：综合分数高于验证集选择的阈值，且短/长动量、趋势、波动、回撤、成交量和流动性过滤同时通过。
- 出场：达到周期持有天数、趋势破坏、综合分数转弱、止损或样本结束强制平仓。
- 验证：阈值只能用训练/验证区间选择，最终样本外区间只评估；滚动前推折用于检查不同时间段的稳定性。
- 输出：样本外净收益、年化收益、年化波动率、Sharpe、Calmar、最大回撤、基准收益/回撤、胜率、交易次数、成本拖累、Brier分数、滚动前推一致性、可信等级和校验和。
- 默认十标的ETF验证池：沪深300ETF、纳指ETF、创业板ETF、中证1000ETF、中证500ETF、证券ETF、光伏ETF、新能源ETF、黄金ETF、国债ETF。
- 通过条件：不能仅看净收益为正；至少要同时检查交易次数、最大回撤、正超额或风险调整比较、滚动前推一致性和概率校准。最终留出通过但跨窗口一致性不足时降级为 WATCH；缺少模拟盘偏差证据时仍不得进入真实下单路径。
- V2 默认增加 10% 趋势效率过滤，避免低效率横盘路径触发入场；该值必须继续通过滚动前推和多资产验证，不能作为永远固定的市场常数。
- V7 对 A 股个股使用资产专属反追涨和市场环境门槛：单日涨幅不超过 3%、21 日动量不超过 15%，且沪深300ETF在信号日可见的 21 日和 63 日动量都不低于 0%。市场代理缺失时不新开仓；ETF不套用该门槛。
- 市场门槛必须逐信号日按时点计算，禁止用当前最终市场状态覆盖历史交易。日线跟踪止损必须先用前一日已确认峰值判断，再更新当日峰值。
- 真实验证至少包含正式、确认、影子三套互不重叠的 A 股池；参数锁定后不得根据确认池和影子池结果再次调参。V7 三套股票池均为 0/10 PASS，当前只能保持研究状态。

#### GUI联动要求

- 顶部策略控件必须直接修改盈利验证入参：周期 `1m/3m/6m/1y` 与每年最大交易次数。
- 当前标的联网行情加载完成后，右侧策略、走势、操作和决策证据面板必须来自 `ProfitBacktestResult`、`ProfitabilityEvidence` 和 `DecisionReport`，不得再用无关占位文案。
- 底部“回测”页必须展示样本外净收益、年化、最大回撤、基准超额、胜率、交易次数、Brier分数和可信等级。
- 即使盈利验证状态为 `PASS`，只要模拟盘成交/偏差证据缺失，最终执行候选仍必须保持 `RESEARCH_ONLY` 或更低等级。
- 手动账户评估只消费当前策略的 `ProfitBacktestResult`、预测和 `DecisionReport`，根据账户资金与持仓计算金额/份额建议；不得创建第二套方向策略，也不得绕过市场、数据、回测和模拟盘门禁。
- QQQ等海外ETF代码可以进入行情链路，但仍使用同一验证和门禁逻辑。

### A股多因子趋势确认

- 证券池：具备流动性、按时点合格的沪深股票；初期研究可限定在主要指数历史成分池。
- 周期：5至20个交易日，不同周期分别建模。
- 因子类别：价值、质量、盈利、投资、动量、相对强弱、低波/下行波动、流动性和市场状态。
- 确认：趋势和价格行为可以作为确认，但不能独立产生高置信度信号。
- 输出：评分、校准概率、收益分位数、最大不利变动估计、操作等级和仓位上限。

### 场外基金分析

- 频率：每周/每月。
- 输入：正式净值收益、回撤、风险调整表现、费用、规模、基金经理/风格稳定性和基准匹配度。
- 不进行日内交易，不使用估算净值回测。

## 预测目标

不同预测周期必须分别训练和评估。候选目标包括：

- 扣除成本后的收益为正/中性/负的概率；
- 达到目标净收益的概率；
- 先触发止损的概率；
- 收益分位数/区间；
- 最大不利变动或回撤分布；
- 跑赢相关基准的概率。

## 基准与模型演进

1. 历史/无条件基准；
2. 简单规则或因子排序；
3. Logistic/线性模型；
4. 在基准通过后再考虑LightGBM、CatBoost等树提升模型；
5. 只有在存在明确增量证据和运维理由时，才考虑更深模型。

## 验证要求

- 按时间顺序划分训练、验证和最终测试集；
- 使用滚动前推评估；
- 对重叠标签使用Purging/Embargo；
- 使用时点证券池和时点披露数据；
- 使用Platt、Isotonic或有证据支持的替代方法进行概率校准；
- 报告Brier分数、对数损失、ECE/校准图、排序指标和组合指标；
- 按年份、市场状态、行业、流动性分组和成本情景评估；
- 执行参数敏感性、多重检验和过拟合诊断。

## 决策逻辑

`BUY_CANDIDATE` 必须同时满足：

- 市场状态可接受；
- 数据和规则有效；
- 可交易且流动性合格；
- 评分/预测高于版本化阈值；
- 扣除成本后的预期收益高于安全边际；
- 下行风险和收益风险比可接受；
- 样本量和概率校准置信度足够；
- 组合仍有可用风险预算。

否则返回 `WATCH` 或 `ABSTAIN`。卖出模型必须独立于“未满足买入条件”。

## 决策中枢MVP输出

TASK-026应交付统一的 `DecisionReport` 或等价领域模型，至少包含以下字段：

- 标的、资产类型、评估时间、数据快照和数据健康状态；
- 当前策略、策略状态、模型版本、规则版本和成本假设；
- 方向概率、收益区间、回撤区间、概率校准状态和样本量；
- 最终建议：买入候选、持有、减仓、卖出、观察或ABSTAIN；
- 建议仓位上限、风险预算占用、止损/止盈参考和失效条件；
- 支持因素、反对因素、关键阈值和历史相似样本摘要；
- 回测与样本外证据：净收益、最大回撤、基准比较、交易次数、成本敏感性和稳定性；
- 模拟盘证据：信号到成交偏差、滑点偏差、漏单/重复信号、运行稳定性；
- 执行候选状态：`NOT_ELIGIBLE`、`RESEARCH_ONLY`、`PAPER_READY` 或 `API_CANDIDATE`。

`API_CANDIDATE` 只表示未来可以进入券商API接入评估，不表示当前版本可以提交真实订单。

## 模型生命周期

`RESEARCH -> VALIDATED -> PAPER_TRADING -> PRODUCTION -> SUSPENDED -> RETIRED`

分别表示：研究、已验证、模拟交易、生产可用、暂停、退役。

提升到生产状态前，必须具备模型卡、数据/版本来源、最终样本外证据、回测审计、模拟交易证据、回滚计划和具名审核决策。模型漂移、校准失败、规则/数据变化或模拟与实际表现偏离时，可以自动暂停模型。
