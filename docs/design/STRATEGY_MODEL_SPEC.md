# 策略与模型规格

## 策略流水线

```text
时点数据
  -> 数据质量门禁
  -> 市场状态
  -> 证券池/可交易性过滤
  -> 指标和因子
  -> 原始策略评分/信号
  -> 校准后的预测分布
  -> 风险与组合约束
  -> 最终操作或ABSTAIN（不交易）
  -> 解释与审计记录
```

参考图：`../../assets/strategy_pipeline.png`。

## MVP策略

### ETF中期轮动

- 证券池：经过批准且流动性良好的宽基、行业/主题、黄金、债券和类现金ETF。
- 频率：每日计算；每周或满足条件时调仓。
- 假设：相对/绝对动量、趋势、波动率控制、分散化和相关性约束。
- 输出：选中资产、目标权重、现金权重、换手率和风险解释。

### A股多因子趋势确认

- 证券池：具备流动性、按时点合格的沪深股票；初期研究可限定在主要指数历史成分池。
- 周期：5至20个交易日，不同周期分别建模。
- 因子类别：价值、质量、盈利、投资、动量、相对强弱、低波/下行波动、流动性和市场状态。
- 确认：趋势和价格行为可以作为确认，但不能独立产生高置信度信号。
- 输出：评分、校准概率、收益分位数、最大不利变动估计、操作等级和仓位上限。

### 场外基金分析

- 频率：每周/每月。
- 输入：正式净值收益、回撤、风险调整表现、费用、规模、基金经理/风格稳定性和基准匹配度。
- 不进行日内交易，不使用估算净值回测。

## 预测目标

不同预测周期必须分别训练和评估。候选目标包括：

- 扣除成本后的收益为正/中性/负的概率；
- 达到目标净收益的概率；
- 先触发止损的概率；
- 收益分位数/区间；
- 最大不利变动或回撤分布；
- 跑赢相关基准的概率。

## 基准与模型演进

1. 历史/无条件基准；
2. 简单规则或因子排序；
3. Logistic/线性模型；
4. 在基准通过后再考虑LightGBM、CatBoost等树提升模型；
5. 只有在存在明确增量证据和运维理由时，才考虑更深模型。

## 验证要求

- 按时间顺序划分训练、验证和最终测试集；
- 使用滚动前推评估；
- 对重叠标签使用Purging/Embargo；
- 使用时点证券池和时点披露数据；
- 使用Platt、Isotonic或有证据支持的替代方法进行概率校准；
- 报告Brier分数、对数损失、ECE/校准图、排序指标和组合指标；
- 按年份、市场状态、行业、流动性分组和成本情景评估；
- 执行参数敏感性、多重检验和过拟合诊断。

## 决策逻辑

`BUY_CANDIDATE` 必须同时满足：

- 市场状态可接受；
- 数据和规则有效；
- 可交易且流动性合格；
- 评分/预测高于版本化阈值；
- 扣除成本后的预期收益高于安全边际；
- 下行风险和收益风险比可接受；
- 样本量和概率校准置信度足够；
- 组合仍有可用风险预算。

否则返回 `WATCH` 或 `ABSTAIN`。卖出模型必须独立于“未满足买入条件”。

## 模型生命周期

`RESEARCH -> VALIDATED -> PAPER_TRADING -> PRODUCTION -> SUSPENDED -> RETIRED`

分别表示：研究、已验证、模拟交易、生产可用、暂停、退役。

提升到生产状态前，必须具备模型卡、数据/版本来源、最终样本外证据、回测审计、模拟交易证据、回滚计划和具名审核决策。模型漂移、校准失败、规则/数据变化或模拟与实际表现偏离时，可以自动暂停模型。
