# 回测规格

## 目标

生成确定性、可审计的模拟结果，尽可能逼近历史每个时点基于当时可获得的信息和规则能够实际执行的交易。

## 事件顺序

```text
MarketEvent（市场事件）
 -> StrategyEvaluation（策略计算）
 -> RawSignal（原始信号）
 -> Data/Rule/RiskDecision（数据/规则/风险决策）
 -> OrderIntent（订单意图）
 -> ExecutionSimulation（执行模拟）
 -> Fill/Reject/PartialFill（成交/拒绝/部分成交）
 -> PortfolioUpdate（组合更新）
 -> Metrics and Audit（指标与审计）
```

## 时间不变量

- 数据只能在其时点可用时间戳之后被观察到。
- T日收盘后计算的信号，不能在同一收盘价成交，除非单独定义的集合竞价/订单流程证明该成交可实现。
- 分钟策略必须定义K线收盘何时可用，以及最早可以执行的后续事件。
- 财务报表和公司事件在实际披露/公告时间之前不可用。
- 调仓、预测和风险决策必须使用同一个时点切片。

## 执行模型

必须提供以下接口：

- `CommissionModel`：佣金模型
- `TaxFeeModel`：税费模型
- `SlippageModel`：滑点模型
- `SpreadModel`：买卖价差模型
- `LiquidityModel`：流动性模型
- `LimitPriceModel`：涨跌停/限价模型
- `LatencyModel`：延迟模型
- `CorporateActionProcessor`：公司行为处理器

成交引擎必须支持拒绝和部分成交。K线价格触及订单价，不代表订单自动全部成交。

## 组合记账

跟踪现金、总股份、可卖股份、平均成本、已实现/未实现盈亏、税费、分红、公司行为新增股份、目标/当前权重、基准价值和资产净值。

## 偏差控制

必须控制：

- 未来数据泄漏；
- 幸存者偏差和历史证券池成员资格；
- 披露日期泄漏；
- 反复使用最终测试集调参；
- 重叠标签泄漏；
- 低估成本和流动性；
- 在涨跌停或停牌状态下假设不可能成交；
- 事后选择证券池。

## 可复现记录

每次运行必须保存：

- 代码提交/构建ID；
- 配置和随机种子；
- 供应商与标准化数据快照ID；
- 证券主数据和规则版本；
- 策略/模型版本；
- 成本/执行模型参数；
- 开始/结束时间以及时区/交易时段定义；
- 结果校验和和环境元数据。

## 必需输出

- 净值/权益曲线和基准；
- 年化/净收益、波动率、最大回撤；
- 适用时的Sharpe、Sortino、Calmar和信息比率；
- 胜率、盈亏比、Profit Factor和期望值；
- 换手率、容量/流动性统计和敞口；
- 交易/订单/成交流水；
- 年度、月度和市场状态拆分；
- 预测型策略的校准和预测诊断；
- 成本/滑点敏感性和参数稳定性。

## ETF容量与冲击成本证据

- ETF组合容量必须复用回测产生的同一调仓事件，不得在容量审计中重选标的或权重。
- 历史ADV只能使用信号日及以前数据；未来成交额变化不得改写过去容量状态。
- 买入和卖出都按目标权重绝对变化计算名义金额。
- 容量报告至少记录账户资金、ADV窗口、目标/硬参与率、最大/P95参与率、最大/P95模型成本、缺失观测和支持资金。
- 交易制度必须按具体ETF产品分类；未知T+0/T+1状态失败关闭。
- 基于日成交额的冲击函数属于压力假设，不替代盘口、队列、价差、部分成交和模拟盘成交证据。
- 账户规模超过目标容量时不得新增或扩大敞口；已有持仓退出仍保留风险优先级。
