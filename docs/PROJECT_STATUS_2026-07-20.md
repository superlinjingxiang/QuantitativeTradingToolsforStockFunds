# 项目近期改动与当前状态（2026-07-20）

## 文档定位

本文汇总当前主线从 FastAPI/Vue/Redis 迁移到 ETF 容量门禁的近期状态。规范性需求仍以 `product-specs/`、`technical/`、`design/` 和 `domain/` 下的事实源为准；策略数字以对应研究报告和运行时数据快照为准。

当前平台仍是量化研究、回测和模拟验证工具，不连接真实券商账户，不执行真实下单，也不承诺未来收益。

## 一、当前运行形态

```text
Vue 3 + TypeScript + Pinia + ECharts
        ↓ localhost JSON
FastAPI + 现有 Python 应用服务
        ↓
行情、预测、单标的策略、ETF组合、回测、DecisionHub、账户评估
        ↓
Redis可选热缓存 + 内存降级 + Parquet历史缓存 + 多数据源
```

- Electron 是默认 Windows 桌面壳，Vue 是默认界面。
- 旧 Electron renderer 和 PySide6 入口继续保留。
- Redis 不可用时项目仍可运行；页面先恢复最近一次成功结果，再后台刷新。
- 上游失败时保留旧数据并明确标记 `STALE/DEGRADED`，旧数据不能升级新的正式交易许可。
- 一键入口仍为 `start_electron.bat`。

## 二、近期用户功能

- 输入代码后回车分析，不要求手动刷新。
- 自选列表和历史浏览记录在本机持久化，重启后恢复。
- 市场指数和荐股池支持启动预取、缓存和失败保留。
- 荐股池只返回普通 A 股账户可买的 A 股、场内 ETF、LOF 和境内跨境 ETF，按评分降序，显示 T+0/T+1。
- 点击荐股候选直接进入同一标的行情、预测和回测链路。
- 手动账户支持计划资金、现金、数量、成本和风险偏好，只消费当前策略最终结果，不产生第二套策略。
- 行情、涨跌幅、成交量、均线、回测信号和预测区间使用同一日期轴；预测区域使用独立虚线、色带和端点。
- 当前价格按中国市场语义显示方向箭头：上涨红色向上，下降绿色向下。
- 深色/浅色主题、整体纵向滚动和右侧紧凑证据卡已完成。

## 三、策略主线

### A 股短线

- 正式版本仍为 `profit-validation-short-v7`。
- 已覆盖目标波动、成本压力、反追涨、次日开盘、T+1、停牌/一字板、止损时点和沪深300市场环境门槛。
- 正式、确认、影子和第四留出四组十 A 股都未形成通用 PASS。
- A 股个股未通过样本外门槛时，不得新增仓位；明确下跌风险仍可对已有仓位给出卖出或减仓研究建议。

### 概率预测

- 当前版本为 `forecast.similar_regime_interval.v2`。
- 21 日预测使用不重叠评估时点并隔离尚未兑现的训练标签。
- 覆盖率、下破率、三分类 Brier 和独立样本数已进入 DecisionHub。
- 历史校准通过不等于策略保证盈利。

### 固定十 ETF 组合

- 当前研究候选为 252 日正绝对动量、每 21 日最多选择 2 只、20% 目标波动、次日开盘执行。
- 15bp 为基础往返成本，45bp 为压力成本。
- 组合证据作为单标的附加门禁进入 `AnalysisReport`、右侧四模块、DecisionHub 和手动账户；不替代单标的方向概率。
- 最后 25% 时间留出仍只有 1 个完整滚动窗口，因此状态为 `WATCH`。

## 四、ETF 容量与冲击成本

2026-07-20 新增 `etf-capacity-impact-v1`：

- 只使用信号日及以前 20 日平均成交额。
- 目标参与率 2%，硬上限 5%。
- 往返成本模型为 `15 + 120 × sqrt(参与率)` bp。
- 缺少成交额、未知交易制度或超过硬门槛时失败关闭。
- 固定十 ETF 的 T+0/T+1 映射复用于荐股标签和组合证据。
- 用户手动账户按计划总资金重新评估容量；资金过大时阻断新增敞口，不把容量不足解释成已有仓位必须清仓。

该模型回答“按近期日成交额估算，账户规模是否明显超过研究容量”，不能回答“真实盘口上能否按该价格全部成交”。

1. A 股短线策略没有跨独立股票池稳定盈利证据。
2. ETF 组合最终留出的完整滚动窗口不足，仍为 `WATCH`。
3. 当前容量模型不是盘口、部分成交或真实冲击证据。

6. 达到样本外、成本、容量、校准和模拟盘门槛前，不评估 `PAPER_READY` 或 API 执行。

## 十、怎样理解当前结果

- “预测校准通过”表示历史概率和区间在当前样本上达到研究门槛，不表示策略一定盈利。
- “容量 PASS”表示给定资金规模没有明显超过基于日成交额的研究上限，不表示盘口一定成交。
- “回测收益为正”仍需同时检查最终时间留出、滚动折、成本压力、容量和模拟盘偏差。
- 当前固定十 ETF 最终时间留出只有 1 个完整滚动窗口，因此保持 `WATCH`。
- 当前 A 股个股多组独立验证仍没有形成通用 PASS，因此不允许把荐股评分直接升级为正式买入许可。
- 手动账户只补充资金、成本和持仓上下文，不能绕过当前策略与 DecisionHub 的门禁。
