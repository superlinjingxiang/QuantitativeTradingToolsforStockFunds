# 项目近期改动与当前状态（2026-07-14）

## 说明范围

本文汇总 2026 年 7 月 11 日至 7 月 14 日的主要工程和策略改动，对应 Git 提交 `049bf94` 之后的当前主线。详细实现记录仍以各日 `CHANGELOG`、策略研究报告、架构文档和代码为准。

当前产品定位没有改变：这是股票与基金的量化研究、回测和模拟验证工具，不连接券商真实账户，不执行真实下单，也不承诺未来收益。

## 一、前后端现代化重构

- 默认桌面界面已迁移为 Vue 3 + TypeScript + Vite + Pinia + ECharts，Electron 继续作为 Windows 桌面壳。
- 本地后端改为 FastAPI，保留 `/api/health`、`/api/search`、`/api/analyze`、`/api/recommendations` 和 `/api/market-overview` 等接口，并自动提供 OpenAPI 文档。
- FastAPI 只负责请求校验和 HTTP 转换，策略、预测、回测、荐股和账户评估继续复用原有 Python 应用服务，没有为新前端复制第二套算法。
- Redis 作为可选热缓存；Redis 不可用时自动降级为进程内缓存，历史 K 线继续使用本地 Parquet 缓存。
- 页面首次打开优先展示最近一次成功数据，再在后台刷新；上游失败时保留上次数据并明确标记 `STALE` 或 `DEGRADED`，陈旧数据不会升级为新交易信号。
- 旧 Electron renderer 和 PySide6 入口仍保留，用于回退和回归检查。

## 二、行情与交互体验

- 支持自动刷新，并避免同一标的重复请求并发覆盖；数据源暂时失败时不清空已经显示的行情。
- 行情图统一展示价格、涨跌幅、成交量、均线、回测信号和预测区间；预测区间使用独立样式，并与历史行情共用日期轴和价格轴。
- 当前价格卡增加涨跌方向和中国市场颜色语义：上涨为红色向上，下降为绿色向下。
- 浅色主题提高文字、表格和控件对比度，图表绘图区在浅色页面下仍使用高对比背景。
- 自选列表和历史浏览记录保存在本机，重新打开软件后可以恢复；点击荐股候选可直接进入对应标的分析。
- 市场指数和荐股池支持缓存及后台预取，界面先展示可用缓存，再无感更新。
- 荐股池仅保留 A 股账户可交易的 A 股、场内 ETF、LOF 和境内跨境 ETF，显示 T+0/T+1，并按评分从高到低排列。

## 三、账户输入与决策联动

- 用户可以按当前标的录入计划总资金、可用现金、持仓数量、成本价和风险偏好，数据只保存在本机。
- 账户评估计算当前市值、持仓成本、浮动盈亏、仓位比例、建议金额和建议份额。
- 账户建议消费当前短线或长线策略的同一份 `DecisionReport`，不维护第二套买卖策略，也不能绕过数据、预测、回测和风险门禁。
- 当前仍未读取券商账户，后续接入真实账户时应通过独立账户适配器转换为现有 `accountContext`，策略输入和决策链保持不变。

## 四、短线策略 V2 至 V7

| 版本 | 主要改动 | 目的 |
|---|---|---|
| V2 | 趋势效率、Sharpe、Calmar、基准回撤、滚动前推一致性 | 降低只看净收益造成的假阳性 |
| V3 | 15bp 基础成本、45bp 成本压力、无交易折计入一致性分母 | 检查成本和跨窗口稳定性 |
| V4 | 20% 目标年化波动、25%-100% 风险暴露、7.5% 跟踪止损、数据快照 | 降低高波动风险并增强复现性 |
| V5 | A 股单日 3% 和 21 日 20% 反追涨约束 | 减少追入短期过热行情 |
| V6 | 收盘信号次日开盘成交、A 股 T+1、停牌和一字涨跌停阻断 | 修正理想成交时点和可交易性偏差 |
| V7 | 前一日确认峰值止损、A 股 21 日反追涨收紧到 15%、沪深300ETF 21/63 日市场门槛 | 消除止损前视并减少逆势开仓 |

V7 的市场门槛只使用信号日及以前可见的数据。市场代理历史缺失时失败关闭，不把缺失信息当作利好。ETF 保持独立交易路径，不误套 A 股个股的 T+1 规则。

## 五、右侧四个决策模块

- **当前策略**：显示短线/长线模式、策略版本、样本窗口、核心指标、市场代理和模型版本。
- **预期走势**：显示上涨、横盘、下跌概率，p05/p50/p95 收益区间、回撤和校准信息。
- **操作与风险**：显示策略建议、等级、仓位上限、不交易原因、失效条件和账户建议。
- **决策证据**：显示研究执行状态，以及数据、成本压力、滚动前推、市场环境、预测校准和模拟盘证据的门禁结果。

四个模块都来自同一份策略、预测、回测和决策结果。`观察`、`仅研究观察` 或 `WATCH` 不是占位文案，而是证据不足或门禁未通过时的真实降级状态。

## 六、V7 真实验证结果

统一条件为 7 年前复权日线、1 个月期限、每年最多 12 次交易、15bp 基础往返成本和 45bp 压力成本。

| 验证池 | PASS | 平均收益 | 中位收益 | 平均最大回撤 | 当前结论 |
|---|---:|---:|---:|---:|---|
| 正式十 A 股 | 0/10 | +1.05% | +3.55% | -10.36% | FAIL |
| 确认十 A 股 | 0/10 | -7.93% | -9.99% | -12.91% | FAIL |
| 影子十 A 股 | 0/10 | +1.23% | -1.49% | -9.79% | FAIL |
| 十 ETF | 2/10 | +18.30% | +17.26% | -10.45% | WATCH |
| 混合十标的 | 2/10 | +8.89% | +2.27% | -10.30% | WATCH |

结论：V7 改善了部分正式样本的收益和回撤，但三个独立 A 股池均为 0/10 PASS，确认池还有明显亏损。当前证据不能证明股票策略具有跨标的稳定盈利能力，因此执行状态继续保持 `RESEARCH_ONLY`。

## 九、当前风险与下一步

1. 股票池结果差异较大，市场过滤还没有形成稳定的跨行业超额收益。
2. 模拟盘成交偏差、滑点漂移、容量和连续运行证据仍不充分。
3. 当前账户输入是手工单标的上下文，不是完整的多资产券商账户同步。
4. Redis 是可选缓存，不是行情数据源；缓存只能改善加载和短时故障体验，不能保证行情实时性。
5. 相对强弱、中期趋势质量和 2% 保本止损已经完成候选研究，但没有通过第四组留出样本和滚动前推门槛；不能因局部回撤改善而晋级。
6. 下一轮应优先完善盘口/容量/部分成交压力、ETF 交易制度分类和模拟盘偏差，并继续保留未参与调参的独立样本。

## 十、V8 候选研究状态

- 增加按时点相对强弱证据和费后保本止损实验能力，但默认配置保持关闭。
- 第四组固定十 A 股留出池中，V7 为 0/10 PASS、平均收益 -2.85%、平均回撤 -12.22%；2% 保本候选同为 0/10 PASS、平均收益 -3.18%、平均回撤 -9.72%。
- 候选虽然改善平均回撤，却降低平均收益、平均 Sharpe、成本压力通过数和滚动正收益比例，因此被拒绝。
- 当前正式版本仍是 V7，项目没有发布一个名义上的 V8 来掩盖不稳定证据。
