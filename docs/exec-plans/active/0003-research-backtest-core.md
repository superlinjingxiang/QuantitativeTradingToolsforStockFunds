# 研究与回测核心执行计划

## 目的和用户可见结果

建立确定性指标/因子、策略接口、事件驱动回测、执行成本、组合风险和报告回归能力。完成后，开发者可以在无真实供应商凭据的情况下，用固定数据快照运行基准策略、生成可复现回测结果，并看到成本、规则、风险和审计证据。

## 背景与仓库定位

TASK-001至TASK-010已完成，项目具备领域契约、数据缓存、质量门禁、市场规则、GUI外壳、搜索切换和首版图表工作区。后续研究/回测核心必须保持无GUI依赖，并使用已有数据、规则和质量服务作为边界。

## 架构与接口

指标、因子、策略、回测、风险、组合和报告模块不得依赖 `ui`。回测必须通过 `MarketRuleEngine` 校验交易约束，通过 `DataQualityService` 阻断坏数据，并用版本化配置和快照ID保证复现。

## 里程碑

### 里程碑1——指标、因子和策略接口

实现TASK-011和TASK-012，建立纯函数指标、因子元数据、策略协议和解释模型。

### 里程碑2——事件回测与执行模型

实现TASK-013和TASK-014，完成事件排序、规则校验、成交/拒绝/费用/流动性和公司行为。

### 里程碑3——组合风险与报告回归

实现TASK-015和TASK-016，完成组合对账、风险门禁、绩效报告、导出和确定性回归夹具。

## 具体实施步骤

从TASK-011开始，先实现指标/因子的版本化注册表与无未来函数校验，再逐步接入策略和回测。

## 验证与验收

每个任务至少运行 `uv run ruff format --check .`、`uv run ruff check .`、`uv run mypy src tests` 和 `uv run pytest`。回测和报告任务还必须加入固定快照回归测试。

## 可复现性、幂等性与恢复

所有夹具必须固定输入、版本、种子和快照ID；报告产物需包含校验和；任务取消不得留下半写入状态。

## 风险与缓解措施

- 未来函数风险：指标和回测必须测试可见时间。
- 成本低估风险：执行模型必须覆盖费用、滑点和流动性。
- 规则绕过风险：策略不得直接下单或跳过规则层。
