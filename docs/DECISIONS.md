### ADR-004——Python 3.12、uv与首轮PySide6依赖
- 日期：2026-06-28
- 状态：已接受
- 背景：TASK-001需要可复现安装、锁文件、测试/类型/代码检查命令，并为后续PySide6桌面GUI保留一致运行时。
- 决策：项目固定Python 3.12，使用 `uv` 和 `pyproject.toml` 管理依赖与锁文件；首轮将 `PySide6` 作为生产依赖加入，但仅做导入冒烟测试，GUI外壳留到TASK-008。
- 已考虑的替代方案：Poetry、pip-tools、延后加入PySide6。Poetry和pip-tools不作为首选；延后PySide6会降低TASK-008前的环境确定性。
- 影响：CI和本地开发统一使用 `uv sync --all-extras --dev`；`uv.lock` 成为可复现安装证据；基础测试覆盖PySide6 `QtCore` 导入。
- 受影响的需求：TASK-001、NFR-08。
- 所需测试/迁移：`ruff format --check`、`ruff check`、`mypy src tests`、`pytest`、`python -m china_quant_platform --version`。

### ADR-005——使用Pydantic实现领域契约模型
- 日期：2026-06-28
- 状态：已接受
- 背景：TASK-002要求标准领域模型、Schema双向转换、时区时间戳校验、概率归一化和可交易报告不变量。
- 决策：使用Pydantic v2实现不可变领域模型，并将 `pydantic` 作为显式生产依赖；所有面向契约的模型提供 `to_contract_dict()` 和 `from_contract_dict()`。
- 已考虑的替代方案：标准库 `dataclasses` 加手写校验、直接使用原始字典。手写校验会重复Schema规则，原始字典不满足有类型领域模型要求。
- 影响：领域层获得统一运行时校验；`DataHealth`、`AnalysisReport`、`BacktestConfig` 和 `SecurityRule` 可与 JSON Schema 往返；后续供应商和GUI必须使用这些模型而不是原始字典。
- 受影响的需求：TASK-002、FR-020、AC-06、AC-10、AC-11。
- 所需测试/迁移：Schema往返测试、非法概率测试、naive时间戳测试、可交易报告来源信息测试、正式/估算基金净值隔离测试。

### ADR-006——使用确定性假供应商验证数据协议
- 日期：2026-06-28
- 状态：已接受
- 背景：TASK-003要求供应商协议不绑定具体SDK，并能在没有真实行情凭据时验证历史、实时、取消和限流行为。
- 决策：在 `data` 层定义 `MarketDataProvider` 协议、请求模型和能力声明；实现 `DeterministicFakeMarketDataProvider` 作为无外部I/O的契约测试供应商。
- 已考虑的替代方案：直接接入真实供应商、使用原始字典夹具。真实供应商会引入凭据和网络不确定性；原始字典不满足有类型契约。
- 影响：后续适配真实供应商必须实现同一协议；领域层继续不依赖 `data` 层或任何供应商SDK；缺能力通过类型化 `DataUnavailable` 暴露。
- 受影响的需求：TASK-003、NFR-06、NFR-08。
- 所需测试/迁移：供应商协议测试、确定性fixture测试、实时订阅测试、取消测试、限流测试和领域层依赖边界测试。

### ADR-007——首版证券主数据采用内存索引
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-004需要本地候选搜索、别名匹配、时点状态和最近搜索；SQLite持久化与GUI切换事务在后续任务中实现。
- 决策：实现 `SecurityMasterService` 作为内存证券主数据服务，使用按日期的 `SecurityMasterRecord` 快照解析时点状态，并用归一化精确/前缀/包含/子序列评分实现本地模糊搜索。

- 影响：搜索路径无外部I/O，适合GUI前的单元/集成测试；后续持久化层可用同一服务接口加载记录。
- 受影响的需求：TASK-004、FR-001、AC-01、NFR-01。
- 所需测试/迁移：T-10搜索测试、时点状态测试、最近搜索测试、2000证券夹具P95搜索性能测试。

### ADR-008——历史K线缓存使用pyarrow写入Parquet分区
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-005要求历史行情按证券、周期和年份保存为Parquet，并且请求前计算缺失区间，只补齐缺口。
- 决策：将 `pyarrow` 作为生产依赖；实现 `HistoricalBarCache` 以单分区文件保存标准 `Bar` 契约行，并由 `MarketDataGateway` 协调缓存读取、缺口请求、幂等追加和实时订阅状态。
- 已考虑的替代方案：CSV/JSON临时缓存、DuckDB优先、Polars优先。CSV/JSON不满足Parquet交付要求；DuckDB跨分区分析留给后续存储查询；Polars对当前单文件读写需求不是必需。
- 影响：历史K线缓存具备可复现的本地列式存储；同一证券、周期、复权和时间戳唯一；实时Quote状态可转换为阻断信号的 `DataHealth`。
- 受影响的需求：TASK-005、FR-003、FR-004、AC-06、NFR-02、NFR-04。
- 所需测试/迁移：Parquet分区读写、重复时间戳、损坏OHLC、缓存幂等追加、增量补缺、陈旧Quote、订阅取消和断线重连测试。

### ADR-009——数据质量门禁输出结构化阻断报告
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-006要求新鲜度、完整性、一致性和授权检查，并保证被阻断数据不得生成最终可交易信号。
- 决策：实现 `DataQualityService`，输出结构化 `DataQualityIssue` 与 `DataQualityReport`，再由报告生成标准 `DataHealth`；阻断状态通过 `assert_signal_allowed()` 映射为类型化 `DataStale`、`DataInvalid` 或 `UnauthorizedData`。
- 已考虑的替代方案：仅在供应商适配器中抛异常、把检查散落在策略层。适配器抛异常难以汇总多项质量问题；策略层分散检查会破坏统一阻断语义。
- 影响：上层策略和GUI可以统一读取健康状态、问题列表和阻断原因；跨源Quote对账可配置容差；历史完整性复用缓存时间窗逻辑。
- 受影响的需求：TASK-006、FR-003、FR-004、FR-020、AC-06、AC-10、C-005。
- 所需测试/迁移：T-08、T-16，重复K线、非法OHLC、缺失K线、陈旧Quote、缺失字段、授权、跨源对账和可交易报告阻断测试。

### ADR-010——首版市场规则使用内存仓库按生效日期解析
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-007要求规则不得硬编码为永久常量，缺失规则时必须关闭式失败，并覆盖中国市场的T+、涨跌停、停牌、费用和基金语义。
- 决策：实现 `InMemoryRuleRepository` 与 `MarketRuleEngine`；规则仍使用标准 `SecurityRule` 契约，交易细节作为审核规则的扩展字段解释；解析时按证券专属优先、交易所、资产类型、生效日期和审核状态选择规则。
- 已考虑的替代方案：直接用证券代码前缀推断规则、现在引入数据库规则表。前缀推断违反规格；数据库持久化留给后续存储/迁移任务。
- 影响：回测和信号层可通过同一规则入口校验订单、价格带、可卖数量和费用；缺失或冲突规则以 `RuleMissing` 关闭式失败。
- 受影响的需求：TASK-007、C-002、C-004、AC-07、AC-08、AC-11。
- 所需测试/迁移：T-02至T-06、T-15，有效期边界、证券专属优先、缺规则失败、T+1、涨跌停流动性、停牌、费用、基金净值语义和披露前不可见测试。

### ADR-011——GUI外壳采用ViewModel状态快照
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-008要求PySide6主窗口、状态模型、导航页签、后台任务抽象和数据健康横幅，同时Qt主线程必须保持响应。
- 决策：实现 `ApplicationViewModel` 持有不可变 `AppUiState` 快照，`MainWindow` 只订阅状态并渲染；首版 `CancellableQtTask` 使用Qt定时器抽象演示非阻塞任务、取消和类型化错误。
- 已考虑的替代方案：直接在Widget里维护业务状态、立即引入QThread池。Widget直管状态会使TASK-009/010难以保证原子切换；QThread池留到真实耗时数据/图表任务接入时再扩展。
- 影响：GUI层获得可测试状态机和健康横幅；后续搜索、订阅和图表更新必须通过ViewModel或应用服务进入状态。
- 受影响的需求：TASK-008、AC-02、AC-06、NFR-03、NFR-06。
- 所需测试/迁移：pytest-qt状态渲染、非阻塞取消和类型化错误状态测试。

### ADR-012——证券搜索与切换事务归属ViewModel
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-009要求搜索控件、键盘操作、`selection_generation` 和旧订阅取消，并且旧证券任务结果不得覆盖新证券状态。
- 决策：`MainWindow` 只负责300ms防抖、候选列表和键盘/鼠标事件；`ApplicationViewModel` 调用 `SecurityMasterService`、记录最近访问、递增 `selection_generation`、取消旧任务并丢弃旧generation结果。
- 已考虑的替代方案：在Widget中直接搜索和选择、使用全局状态单例。Widget直连业务难以测试竞态；全局状态会破坏多窗口/测试隔离。
- 影响：后续历史加载、实时订阅、图表刷新和策略计算必须携带选择代次，避免旧代结果覆盖当前上下文。
- 受影响的需求：TASK-009、FR-001、FR-002、AC-01、AC-02、NFR-06。
- 所需测试/迁移：T-07/T-10，GUI防抖搜索、键盘候选选择、选择代次、旧任务取消和旧generation结果丢弃测试。

### ADR-014——首版指标与因子使用纯Python确定性计算
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-011要求指标是纯函数或可缓存的确定性计算，必须处理预热期、缺失值和未来函数风险；当前仓库尚未引入pandas/numpy作为运行时依赖。
- 决策：在 `indicators` 中用标准库实现SMA/EMA/MACD/RSI/ATR/布林带/VWAP、收益、波动、下行波动、回撤和相对强弱；在 `factors` 中实现版本化 `FactorMetadata`、`FactorRegistry` 和默认点时因子。缓存键使用排序JSON与SHA256生成。
- 已考虑的替代方案：立即引入pandas或TA-Lib、把指标直接写在策略层。pandas/TA-Lib会增加依赖和安装面；策略层内嵌指标会削弱复用和审计。
- 影响：指标和因子不依赖GUI、供应商或真实数据库；后续策略/回测可以复用同一组点时计算与缓存键。
- 受影响的需求：TASK-011、FR-012、AC-10、NFR-08。
- 所需测试/迁移：预热期、缺失值、未来值隔离、因子元数据、重复注册、默认因子计算和缓存键排序无关性测试。

### ADR-015——策略层只输出原始信号与解释
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-012要求策略协议、原始信号、适用/失效条件和解释，同时明确策略不得直接产生最终订单或绕过风险与规则层。
- 决策：在 `strategies` 中定义 `Strategy` 协议、`StrategyContext`、`WarmupSpec`、`StrategyMetadata`、`RawSignal`、`Explanation` 和 `evaluate_strategy()`；`RawSignal` 使用独立 `RawSignalIntent`，强制 `requires_data_quality_gate`、`requires_rule_gate` 和 `requires_risk_gate` 为真，并禁止订单/最终信号字段。
- 已考虑的替代方案：让策略直接输出 `FinalSignal` 或目标仓位、把解释作为自由文本。直接输出最终决策会绕过规则/风险流水线；自由文本解释难以审计和测试。
- 影响：后续回测和风控层可以把策略输出作为受控输入，继续在数据、规则、风险和执行层生成订单意图、成交和最终信号。
- 受影响的需求：TASK-012、FR-006、FR-018、AC-04、AC-10。
- 所需测试/迁移：RawSignal越权字段拒绝、门禁不可关闭、预热不足阻断、策略/证券/快照身份一致性和解释一致性测试。

### ADR-016——回测内核先固定事件时序与规则拒绝语义
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-013要求事件循环、交易时钟、订单/成交/拒绝/部分成交、取消和审计；成本、滑点、流动性容量和公司行为将在TASK-014细化。
- 决策：实现 `BacktestEventLoop` 以时间、事件优先级和序列排序；`BacktestEngine` 只在市场事件后评估策略，订单必须晚于信号时间，并通过 `MarketRuleEngine` 校验后生成 `FILL`、`PARTIAL_FILL` 或 `REJECT` 事件。运行结果用事件快照SHA256作为checksum。
- 已考虑的替代方案：在TASK-013直接实现完整投资组合/成本/滑点、或用线性for循环替代事件队列。完整执行模型会扩大任务边界；线性循环难以处理跨日订单、取消和后续异步事件。
- 影响：TASK-014可在同一执行模拟器位置接入费用、滑点、流动性和公司行为；TASK-016可复用事件checksum和审计记录做回归。
- 受影响的需求：TASK-013、FR-013、AC-07、AC-09、NFR-05、NFR-08。
- 所需测试/迁移：事件优先级、策略时序、订单时间防泄漏、规则拒绝、部分成交、取消和重复运行checksum一致测试。

### ADR-017——执行模型以可组合小模型接入回测模拟器
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-014要求佣金、税费、滑点、价差、流动性、涨跌停、延迟和公司行为；其中涨跌停/停牌/手数仍应由市场规则引擎统一负责。
- 决策：新增 `backtest.execution`，提供 `RuleBasedCostModel`、`FixedBpsSlippageModel`、`FixedSpreadModel`、`ParticipationRateLiquidityModel`、`FixedLatencyModel` 和 `CorporateActionProcessor`；`DeterministicExecutionSimulator` 组合这些模型并继续调用 `MarketRuleEngine` 做规则校验。
- 已考虑的替代方案：把所有执行逻辑写进 `BacktestEngine`、或现在引入真实券商/盘口模型。集中写入引擎会难以替换和测试；真实盘口模型需要数据契约和供应商接入，超出当前任务。
- 影响：后续可在不改策略接口的情况下替换更真实的滑点、盘口和容量模型；组合对账可复用公司行为处理器。
- 受影响的需求：TASK-014、AC-08、AC-11、T-02至T-06、T-15、T-18。
- 所需测试/迁移：费用/税费、滑点/价差、参与率部分成交、延迟、涨停无对手方流动性拒绝、分红和拆股测试。

### ADR-018——组合与风险引擎分离记账和门禁
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-015要求现金、持仓、可卖数量、仓位计算、集中度、相关性、流动性和回撤门禁；这些能力既服务回测，也服务模拟账户和信号解释。

### ADR-022——预测输出必须是概率、区间和可拒绝状态
- 日期：2026-06-29
- 状态：已接受
- 背景：TASK-019要求经过校准的预测引擎、收益分位数和分布外、漂移、样本不足时的不交易结果，且AC-05禁止暗示确定收益。
- 决策：在 `forecasting` 中实现 `ForecastEngine`、`ForecastConfig`、`LogisticCalibrator` 和校准指标函数；预测结果始终以方向概率、收益分位数和期望回撤表达，样本不足、分布外、漂移或低置信度时返回 `ABSTAIN` 并只把概率作为诊断估计。
- 已考虑的替代方案：直接输出点预测收益、或由策略层自行决定是否不交易。点预测容易被误解为收益承诺；策略层分散不交易规则会削弱统一审计和GUI呈现。
- 影响：TASK-020可以把同一预测结果接入分析报告和GUI面板；后续模型替换必须继续提供校准指标和不交易原因。
- 受影响的需求：TASK-019、FR-007、AC-05、AC-10、AC-12。
- 所需测试/迁移：READY概率/分位数测试、样本不足/分布外/漂移/低置信度不交易测试、Brier/LogLoss/ECE校准指标测试。
